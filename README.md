# 工数一覧からガントチャートを自動生成するツール

担当者・工数(日)・優先順位を記入したExcel(B)から、土日・祝日・年末年始・会社独自の
休日を考慮したガントチャート付きExcel(C)を自動生成します。出力したCを次回のBとして
そのまま使い回せるため、進捗管理を兼ねたガントチャートとして運用できます。

- 依存関係はなし。**優先順位の順に**、**担当者の空き状況**を見ながら
  タスクを詰めて配置します(同じ担当者は同時に1タスクのみ、別の担当者は並行可能)。
- 各タスクの開始日は、**「担当者」シートに記載したその担当者の参画開始日**を
  下限として計算されます。後から参画するメンバーがいても、そのメンバーの参画開始日を
  指定するだけで、その日より前には日程が組まれないようにできます。
- 土日・日本の祝日・年末年始(12/27〜1/3)・会社独自の休日は作業日から除外されます。
  これらはすべて「休日」シートに**日付として記録**され、Excel上でこのシートを直接
  編集すれば(行を追加・削除すれば)、ガントチャート側の表示もそれに追従します。
- 「分類」「作業Lv1」「作業Lv2」などBに追加した列はそのままCにコピーされます。
- 「進捗(%)」を手入力して再実行すれば、その値を保持したまま日程だけ再計算されます。
- タイムラインのバー・休日の塗り・今日の縦線はすべて**条件付き書式**で表示されます。
  Excel上で開始日・完了日・担当者シート・休日シートを直接編集すると、表示もExcel側で
  自動的に追従します。
- 同じBファイルから何度でもCを作り直せます(Bを直してスクリプトを再実行するだけ)。
- ネットワーク通信は祝日データの初回取得時のみ。それ以外は完全にローカルで動作します。

## 必要なもの

- Node.js (動作確認済みバージョン: v22。v18以降であれば動作する見込みです)
- インターネット接続は **初回の祝日データ取得時のみ** 必要です。職場PCがネットワークに
  繋がらない場合は、後述の「祝日データの準備」を参照してください。

## セットアップ

```bash
npm install
```

依存パッケージ(exceljs, iconv-lite)がインストールされます。
ネットに繋がらない職場PCでは `npm install` 自体が実行できないため、
**自宅PCなどネットに繋がる環境で `npm install` まで済ませた状態のフォルダ全体
(node_modules含む)をGitHubに上げて、職場でそのままcloneして使う**ことを想定しています。
(`.gitignore` で `node_modules` を除外していますが、職場で `npm install` できない場合は
`.gitignore` から `node_modules/` の行を削除してコミットしてください。)

## 祝日データの準備 (重要・最初に1回だけ)

このツールは内閣府が公開している祝日CSV
(`https://www8.cao.go.jp/chosei/shukujitsu/syukujitsu.csv`)から祝日データを取得し、
`holidays_cache.csv` というファイルにキャッシュします。

- **キャッシュファイル(`holidays_cache.csv`)が既にあれば、ネットワークには一切アクセスしません。**
- キャッシュがない状態で実行すると、自動でダウンロードを試みます。

職場PCはネットワークに繋がらないため、**ネットに繋がる環境(自宅PCなど)で一度
このツールを実行して`holidays_cache.csv`を作成し、それをこのフォルダに置いた状態で
GitHubにコミットしてください。** 職場ではそのキャッシュファイルがそのまま使われます。

```bash
# ネットに繋がる環境で一度だけ実行(キャッシュ作成のため)
node make-gantt.js サンプル.xlsx
```

実行すると `holidays_cache.csv` が生成されます。これをコミットして職場に持ち込めば、
職場PCではネットワークアクセスなしで動作します。

### キャッシュの更新について

内閣府CSVは確定した祝日のみを含むため、何年も先の祝日(春分の日・秋分の日など)は
含まれていないことがあります。年が変わったら、ネットに繋がる環境で
`holidays_cache.csv` を削除してから再実行し、最新版を取り直すことをおすすめします。

```bash
rm holidays_cache.csv
node make-gantt.js サンプル.xlsx
```

## 文字化けする場合

実行中の質問文や案内メッセージが文字化けする場合、ターミナル側の文字コード設定が
原因であることがほとんどです(このツール自体はUTF-8で出力しています)。
以下を試してください。

- **PowerShellの場合**: 実行前に `chcp 65001` (UTF-8への切り替え)を一度実行する。
- **コマンドプロンプト(cmd.exe)の場合**: 同様に `chcp 65001` を実行するか、
  Windows Terminalから実行する。
- **VS Codeの統合ターミナルの場合**: ターミナルの種類をPowerShellまたはGit Bashに
  切り替えてみる。設定の `terminal.integrated.profiles.windows` で確認・変更できます。

## 使い方

### 1. Bファイル(入力Excel)を準備する

Bファイルには **2つのシート** が必要です。

**シート1: タスク一覧** (シート名は自由。1枚目として置いてください)

1行目をヘッダーとし、以下の列を含めてください(列の順番は自由、他の列を追加しても
構いません。追加列もそのままC側にコピーされます)。

| 分類 | 作業Lv1 | 作業Lv2 | タスク名 | 担当者 | 工数(日) | 優先順位 |
| ---- | ------- | ------- | -------- | ------ | -------- | -------- |
| 開発 | 計画    | 要件    | 要件定義 | 田中   | 5        | 1        |
| 開発 | 計画    | 設計    | 基本設計 | 田中   | 8        | 2        |
| 開発 | 実装    | 設計    | 詳細設計 | 佐藤   | 6        | 3        |

- 必須列は **タスク名・担当者・工数(日)・優先順位** の4つだけです。
  「分類」「作業Lv1」「作業Lv2」やその他の列は自由に追加・削除できます。
- **優先順位**: 数字が小さいほど先に着手されます。同じ数字の場合は表の上から順に
  処理されます(ただし表示順には影響しません。後述)。
- **工数(日)**: 土日・祝日を除いた実働日数で指定してください。
- 「進捗」列(0〜100の数値、%表記)を入れておくと、その値を引き継いで出力されます。
  ない場合は全タスク進捗0%として扱われます。
- 列名は多少のゆらぎを許容します(例: 「工数」「日数」でも「工数(日)」として認識されます)。
  認識されない場合はエラーメッセージで知らせます。

**シート2: 「担当者」** (シート名は必ず「担当者」にしてください)

| 担当者名 | 参画開始日 |
| -------- | ---------- |
| 田中     | 2026/6/22  |
| 佐藤     | 2026/6/22  |
| 鈴木     | 2026/7/1   |

- タスク一覧に登場する担当者は **全員このシートに記載が必要です。** 記載がない
  担当者がいる場合はエラーになります。
- 「参画開始日」はその担当者がプロジェクトに着手できる最初の日です。後から
  参画するメンバーがいる場合、ここに参画予定日を入れておけば、その日より前には
  タスクが割り当てられません。
- プロジェクト全体の「開始日」を別途指定する必要はありません。各担当者の
  参画開始日の組み合わせで自動的に決まります。

**シート3: 「休日」 (省略可・任意)**

会社独自の休日(創業記念日など)を追加したい場合、「休日」という名前のシートを
作り、以下の列を用意してください。なくても動作します(ツールが自動生成します)。

| 日付      | 名称       |
| --------- | ---------- |
| 2026/8/15 | 創業記念日 |

このシートは**Cにも必ず出力され**、内閣府の祝日・年末年始休暇も自動で追記されます。
**Excel上でこの「休日」シートの行を追加・削除するだけで、ガントチャートの休日表示が
動的に変わります**(再実行は不要です。条件付き書式がこのシートを直接参照しているため)。

### 2. スクリプトを実行する

```bash
node make-gantt.js B.xlsx
```

実行すると、ガントチャートの粒度(日単位/週単位)を対話形式で聞かれます。
何も入力せずEnterを押すと週単位になります。

コマンドライン引数で直接指定することもできます(繰り返し実行する場合に便利です):

```bash
node make-gantt.js B.xlsx --granularity day
```

| オプション            | 説明                                |
| --------------------- | ----------------------------------- |
| `--granularity`, `-g` | `day`(日単位) または `week`(週単位) |

### 3. 出力(C)を確認する

Bファイルと同じフォルダに `gantt_YYYYMMDD_HHMMSS.xlsx` という名前でガントチャートが
出力されます。実行するたびに新しいファイルが作られるので、古いファイルは上書きされません。

Cには3つのシートが含まれます。

- **「ガントチャート」**: メインのシート。列構成は「Bの全列 + 開始日 + 完了日 + 進捗(%)」
  になります(進捗列は常に完了日の直後に配置されます)。
- **「休日」**: 祝日・年末年始・会社独自の休日の一覧(日付・名称)。
- **「担当者」**: 担当者ごとの参画開始日。次回Bとして再利用する際に読み込まれます。

ガントチャートシートの表示について:

- タイムラインのヘッダーは「月」(1行目) + 「日付」(2行目、日単位ではさらに3行目に「曜日」)
  の構成です。月は1行目にまとめて表示されるため、2行目に重複しては表示されません。
  日付は日付型のセルなので、Excel上でそのまま日付として扱えます。
- タイムラインの範囲は、最初のタスクが属する月の1日から、最後のタスクの完了予定月の
  **1ヶ月後の月末**までになります(余裕を持たせるため)。月初〜月末で揺れなく表示されます。
- 日単位モードでは土日・「休日」シート掲載日の列がグレーで塗られます。これは
  `WEEKDAY()`関数と`COUNTIF()`関数による条件付き書式で実現しているため、
  「休日」シートを編集すれば即座に表示が変わります。**休日表示は常に最優先されるため、
  開始日・完了日をどう変更しても休日列が作業中のように見えることはありません。**
- 作業期間は青いバーで表示されます。**このバーは条件付き書式で実現しているため、
  Excel上で開始日・完了日のセルを直接編集すると、バーの表示もその場で追従します。**
- 今日の列の左側には赤い縦線が引かれます。これは`TODAY()`関数で判定しているため、
  ファイルを開いた日が常に「今日」として扱われます(ファイル作成日には固定されません)。
- **タスクの表示順は常にBファイルの行順をそのまま保持します。** 優先順位や担当者の
  空き状況は日程計算(開始日・完了日の算出)にのみ使われ、表示順には影響しません。

### 4. 進捗を更新しながら何度も作り直す

このツールの特徴は、**出力したCをそのまま次回のBとして使い回せる**ことです。

1. 出力された `gantt_*.xlsx` を開き、「進捗(%)」列に手で進捗を入力する
   (例: 50%進んでいたら `50` と入力)。
2. 必要なら「担当者」シートの参画開始日や「休日」シートの休日も編集する。
3. そのファイルをそのまま `node make-gantt.js gantt_*.xlsx` のように指定して再実行する。
4. 「開始日」「完了日」列は無視され、優先順位と担当者の参画開始日・空き状況から
   日程が再計算されます。一方、進捗(%)・担当者の参画開始日・休日シートの内容
   (会社独自の休日を含む)はそのまま引き継がれます。

これを繰り返すことで、進捗管理を兼ねたガントチャートとして運用できます。

もちろん、最初のBファイルに新しいタスクを追加したり、担当者・工数・優先順位を
直接編集して再実行することもできます。

## ファイル構成

```
make-gantt.js       メインスクリプト(実行する対象)
src/holidays.js      祝日データの取得・キャッシュ
src/readTasks.js     Bファイル(または前回出力したC)の読み込み(タスク・担当者・休日)
src/scheduler.js     優先順位+担当者の参画開始日・空き状況によるスケジューリング
src/buildGantt.js    Cファイル(ガントチャートExcel)の生成
holidays_cache.csv   祝日データのキャッシュ(初回ネット接続時に生成。要コミット)
```

## GitHub経由で職場PCに持ち込む手順(例)

1. ネットに繋がる環境(自宅PCなど)でこのフォルダを使い、`npm install` と
   「祝日データの準備」を済ませる(`holidays_cache.csv` ができている状態にする)。
2. GitHubに新しいリポジトリを作成し、このフォルダの内容をpushする。
   ```bash
   git init
   git add .
   git commit -m "init"
   git remote add origin <あなたのリポジトリURL>
   git push -u origin main
   ```
   `node_modules` を含めてpushしたい場合は `.gitignore` から該当行を削除してください
   (職場PCで `npm install` できない場合はこちらが必要です)。
3. 職場PCで `git clone <リポジトリURL>` する。
4. `node make-gantt.js B.xlsx` を実行する。

## 注意事項

- 内閣府の祝日CSVのURLは過去に「syukujitsu.csv」⇔「shukujitsu.csv」で変更されたことが
  あります。このツールはメインURL(`syukujitsu.csv`)が失敗した場合に自動で
  `shukujitsu.csv`も試しますが、両方変更されてしまった場合は
  `src/holidays.js` 内の `HOLIDAY_CSV_URL` を内閣府サイトで確認した最新URLに
  書き換える必要があります。
- 機密情報を含むBファイル・Cファイルはこのリポジトリにはコミットしないでください
  (`.gitignore` で `gantt_*.xlsx` と `test_*.xlsx` は除外していますが、
  実際の業務データのファイル名がそれに一致しない場合は注意してください)。
